## 一、本周工作完成情况

### 1. 订单查询接口性能优化

- [x] 为 `orders` 表添加联合索引 `(user_id, status, created_at)`
- [x] 优化 SQL 查询语句，减少全字段查询，增加条件过滤与分页
- [x] **查询耗时从平均 320ms 降至 45ms，性能提升 86%**

**优化前 SQL：**

```sql
SELECT * FROM orders
WHERE user_id = ?
ORDER BY created_at DESC;
```

**优化后 SQL：**

```sql
SELECT id, order_no, status, pay_amount, created_at
FROM orders
WHERE user_id = ? AND status IN (1, 2, 3)
ORDER BY created_at DESC
LIMIT 20;
```

> 📌 优化要点：避免 `SELECT *`、增加 `status` 过滤条件、使用 `LIMIT` 限制返回行数，使查询能够命中覆盖索引。

🔗 PR：[#456 订单查询优化](https://github.com/company/backend/pull/456)

### 2. 用户头像上传 OOM Bug 修复

- [x] 定位根因：上传图片未做大小限制，直接全量读入内存导致 **OOM**
- [x] 增加文件大小校验（**最大 5MB**）
- [x] 使用 `Pillow` 库进行图片压缩与 Resize 处理

**核心修复代码：**

```python
img = Image.open(file)
if max(img.size) > 1920:
    img.thumbnail((1920, 1920))
img.save(output, quality=85)
```

🔗 PR：[#461 头像上传修复](https://github.com/company/backend/pull/461)
